# Devhost

Devhost routes local development hostnames to local HTTP ports. It is HTTP-only and uses dnsmasq for optional wildcard local DNS on macOS.

## Quick Start

Build Devhost, then run the proxy on port `80` with DNS setup:

```sh
cargo build
sudo target/debug/devhost serve --config devhost.toml --setup-dns
```

In another terminal, test an unmatched host:

```sh
curl -i -H "Host: unknown.test" http://127.0.0.1/
```

Expected result: `404 Not Found`.

Test a configured host:

```sh
curl -i -H "Host: agency.test" http://127.0.0.1/
```

`agency.test` routes to `http://127.0.0.1:2000`, as configured in `devhost.toml`. If an app is running on port `2000`, its response is returned. If nothing is listening there, Devhost returns `502 Bad Gateway`.

Then open the clean URL in a browser:

```txt
http://agency.test/
```

## Config

```toml
listen = "127.0.0.1:80"

[dns]
tld = "test"
loopback_ip = "127.0.0.1"
dnsmasq_config_path = "/opt/homebrew/etc/dnsmasq.d/devhost.conf"
dnsmasq_conf_path = "/opt/homebrew/etc/dnsmasq.conf"
resolver_path = "/etc/resolver/test"

[[routes]]
host = "agency.test"
target = "http://127.0.0.1:2000"
```

## Commands

```sh
cargo run -- validate --config devhost.toml
cargo run -- routes --config devhost.toml
cargo build
sudo target/debug/devhost serve --config devhost.toml
cargo run -- doctor --config devhost.toml
```

## DNS Setup

Preview the files Devhost would manage:

```sh
cargo run -- install-dns --config devhost.toml --dry-run
```

Install the DNS integration:

```sh
sudo target/debug/devhost install-dns --config devhost.toml
sudo brew services restart dnsmasq
```

You can also bundle DNS setup into `serve`:

```sh
cargo build
sudo target/debug/devhost serve --config devhost.toml --setup-dns
```

That command runs the privileged DNS installer first, restarts dnsmasq, and then starts the normal proxy.

On macOS, dnsmasq must run as a root service to listen on DNS port `53`, so Devhost restarts it with:

```sh
sudo brew services restart dnsmasq
```

Devhost writes:

```txt
/opt/homebrew/etc/dnsmasq.d/devhost.conf
/opt/homebrew/etc/dnsmasq.conf
/etc/resolver/test
```

It only removes files or marked config blocks generated by Devhost:

```sh
sudo target/debug/devhost uninstall-dns --config devhost.toml
sudo brew services restart dnsmasq
```

With `listen = "127.0.0.1:80"`, DNS lets you use clean local URLs:

```txt
http://agency.test/
```

Port `80` is privileged on macOS, so the proxy must be started with sudo or later installed as a service.

## Testing With `/etc/hosts`

For exact hostnames, add entries like this:

```txt
127.0.0.1 agency.test
```

Then browse to:

```txt
http://agency.test/
```

`/etc/hosts` does not support wildcards. Wildcard domains like `*.agency.test` need the dnsmasq setup above.
